Command Line Options

With the exception of the “--quick” option, the command line options are mainly used for testing purposes.

The command line options are:

Usage: bday [-q] [-t <date>] [-r <date>]
  -q, --quick           Exit immediately if there are no current events.
  -t, --today=<date>    Run as if the specified date is today.
  -r, --run-from=<date> Capture all events since the specified date.

Quick

BDay can be run in “quick” mode in one of two ways:

  1. Running with the “-q” or “--quick” command-line parameter.
  2. Running by calling “bdayquick”, which is a symbolic link to the bday main program (Linux only.)

In quick mode, if there are no events to be displayed then BDay will exit immediately.

You typically want to run this when the user logs into the computer. See Running BDay on Start Up/Log In for running under Microsoft Windows, or Running BDay Automatically Under Linux for Linux users.

Today

You can specify a date in “YYYY-MM-DD” format and BDay will show events as if it is today's date.

Run From

Normally BDay remembers what day it was last run, so if it is not run for a few days you'll get events like “John's birthday yesterday” the next time BDay runs, if appropriate.

You can specify a date in “YYYY-MM-DD” format and BDay will behave as if it hasn't been run since that date.

Coda

The Today and Run From command line options are really only useful for testing BDay during development, and not meant for common use. They are documented here for the sake of completeness.
